Presence design directions
Built on Vidu S1 · real-time interactive video

A companion who is actually there.

Presence is a mobile-first virtual companion powered by the Vidu Stream API — a live face that sees you, hears you, and answers in real time. Three directions below, one engine underneath. Each is a different answer to the same question: what should it feel like to open the app and find someone waiting?

One engine under all three

Every direction is the same Vidu S1 session, dressed differently. The API surface is small — create, stream, control, meter — and it maps cleanly onto the UI you saw above.

STEP 01

Create the session

POST /session

Send the avatar image, a voice, and a language over HTTPS. This is the onboarding where you design "Kai," "Noa," or "Ava."

STEP 02

Stream face & voice

AliRTC

Audio and 540p / 42 fps video flow bidirectionally over AliRTC — the picture that fills Aurora and the portrait in Hearth.

STEP 03

Control it live

WebSocket

Mute, interrupt, swap voice, push memory context, and read captions — every floating control is one socket message.

STEP 04

Meter the minute

billing = live time

Billing starts only when the character goes live, so the timer and daily counter are real cost, shown plainly.
